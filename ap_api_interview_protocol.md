# AP API (Awesome Person API)

## Interview Assessment Protocol & Pattern-Based Evaluation Framework

**Version 1.0 | January 2026** **Author: Kevin Keller, Tucuxi Inc.**

---

## Executive Summary

The AP API is a systematic approach to assessing candidate soft skills and personality characteristics through structured interviews grounded in cognitive science principles. Rather than relying on subjective impressions or easily-gamed self-reports, the system uses **behavioral archaeology**—excavating specific past instances to reconstruct patterns of thinking, adaptation, and communication.

The framework operationalizes reasoning patterns from the Universal Reasoning Pattern (URP) library to transform tacit assessment skills into explicit, teachable, and measurable protocols.

---

## Part 1: Core Philosophy

### The Problem with Traditional Assessment

Traditional interviews suffer from several systematic failures:

1. **Self-Report Bias**: Candidates describe who they *want* to be, not who they *are*  
2. **Interviewer Confirmation Bias**: First impressions shape interpretation of subsequent evidence  
3. **Halo/Horns Effects**: Single strong/weak signals color overall assessment  
4. **Cultural Pattern Matching**: Interviewers favor candidates who "feel" similar  
5. **Memory Distortion**: Both parties misremember what was actually said

### The AP API Solution

The AP API addresses these failures through:

| Failure | Solution | Underlying Pattern |
| :---- | :---- | :---- |
| Self-report bias | Behavioral archaeology (dig into specifics) | MC24 \- Assumption Surfacing |
| Confirmation bias | Structured hypothesis testing | MC44 \- Solution Space Exploration |
| Halo/horns effects | Multi-dimensional scoring rubrics | MC35 \- Representation Choice |
| Cultural matching | Explicit criteria, blind calibration | IP11 \- Trust Calibration |
| Memory distortion | Real-time scoring, recorded interviews | MC29 \- Progress Monitoring |

The following URP patterns form the core assessment methodology:

## Part 3: The STAR+ Framework

### Enhanced Behavioral Interview Structure

The AP API extends the standard STAR framework (Situation, Task, Action, Result) with two additional components:

STAR+ Framework:

  S \- Situation: "Tell me about a time when..."

  T \- Task: "What was your specific role/responsibility?"

  A \- Action: "What specifically did YOU do?" (not the team)

  R \- Result: "What happened?"

  

  \+ Reflection: "What would you do differently?"

  \+ Recursion: "Tell me about another time..." (verify consistency)

### Why Reflection Matters

The **Reflection** component (+R) is where true assessment occurs:

| Response Type | Signal | Interpretation |
| :---- | :---- | :---- |
| Specific alternative identified | Strong self-awareness | Adaptability present |
| "Nothing, it worked out" | Low reflection | Adaptability questionable |
| Defensive justification | Ego protection | May resist feedback |
| "Everything was wrong" | Over-critical | May lack confidence |
| Nuanced trade-off analysis | High metacognition | Strong growth potential |

### Why Recursion Matters

The **Recursion** component (+R) tests pattern consistency:

- Same trait, different context → Is this a real pattern?  
- If stories contradict → Which is accurate?  
- If examples are thin → Is this a growth area?

## Part 4: Trait Assessment Protocols

### 4.1 Curiosity Assessment Protocol

**Definition**: The intrinsic drive to explore, understand, and learn beyond immediate requirements.

**MC24 Application**: Surface assumptions about what "curiosity" means in this role context.

#### Primary Probes

| Probe | Pattern | What to Listen For |
| :---- | :---- | :---- |
| "What's something you changed your mind about recently?" | MC24 | Curious people have a LIST of these |
| "Walk me through how you approach something you don't understand." | MC44 | Process reveals disposition |
| "What have you learned recently that wasn't required for work?" | Direct | Voluntary learning signals intrinsic motivation |
| "What questions do you have for me?" | IP3 | Curious people can't help asking questions |

#### Behavioral Indicators (Observable)

| High Curiosity | Low Curiosity |
| :---- | :---- |
| Asks clarifying questions throughout | Waits passively for information |
| Connects topics across domains | Stays narrowly focused |
| Admits knowledge gaps eagerly | Avoids admitting uncertainty |
| Challenges interviewer assumptions | Accepts all premises |
| Provides examples of voluntary learning | Learning tied to requirements |

#### Scoring Rubric

| Score | Description |
| :---- | :---- |
| **5** | Demonstrates active, self-directed learning across multiple domains; asks probing questions; shows evidence of changing views based on new information |
| **4** | Shows consistent curiosity within professional domain; asks good questions; can cite recent learning |
| **3** | Average curiosity; learns when prompted; asks clarifying questions but not exploratory ones |
| **2** | Limited curiosity; learning is primarily reactive; few questions asked |
| **1** | No evidence of curiosity; passive stance toward information; dismisses unfamiliar topics |

### 4.2 Adaptability Assessment Protocol

**Definition**: The capacity to adjust approach when circumstances change or initial strategies fail.

**IP7 Application**: Explore conflict and failure—adaptability reveals itself under stress.

#### Primary Probes

| Probe | Pattern | What to Listen For |
| :---- | :---- | :---- |
| "Tell me about a time your approach wasn't working. What happened?" | IP7 | Speed and quality of pivot |
| "How do you know when to pivot vs. persist?" | MC29 | Metacognitive monitoring |
| "Describe a situation where you had to learn something completely new quickly." | STAR+ | Learning velocity |
| "If you joined and discovered everything you assumed was wrong, what would you do?" | SP12 | Hypothetical adaptability |

#### Behavioral Indicators (Observable)

| High Adaptability | Low Adaptability |
| :---- | :---- |
| Multiple examples of course correction | Stories lack pivots |
| Describes discomfort but engagement | Avoids discomfort narratives |
| Uses failure as learning data | Frames failures as external |
| Comfortable with hypotheticals | Resists hypothetical scenarios |
| Changes communication style with interviewer | Static interaction style |

#### Scoring Rubric

| Score | Description |
| :---- | :---- |
| **5** | Multiple clear examples of rapid, effective pivots; treats change as opportunity; demonstrates real-time adaptability in interview |
| **4** | Good examples of adaptation; reasonable pivot speed; comfortable with ambiguity |
| **3** | Can adapt when required; prefers stability; pivots with support |
| **2** | Slow to adapt; requires significant evidence before changing approach; uncomfortable with ambiguity |
| **1** | Rigid; resists change; failures attributed externally; no evidence of adaptation |

### 4.3 Communication Assessment Protocol

**Definition**: The ability to convey information clearly, calibrate to audience, and listen effectively.

**MC35 Application**: Can they shift representation based on audience?

#### Primary Probes

| Probe | Pattern | What to Listen For |
| :---- | :---- | :---- |
| "Explain \[technical concept from their background\] to me as if I know nothing about it." | MC35 | Calibration to non-expert |
| "Now explain it to me as if I'm an expert." | MC35 | Calibration to expert |
| "Who was the hardest person to communicate with and why?" | MC1 | Stakeholder awareness |
| "Tell me about a time you had to deliver difficult feedback." | IP7 | Conflict communication |

#### Behavioral Indicators (Observable)

| Strong Communication | Weak Communication |
| :---- | :---- |
| Checks for understanding | Monologues without checking |
| Adjusts complexity based on feedback | Same explanation regardless of cues |
| Structures information clearly | Disorganized, circular |
| Listens before responding | Interrupts or waits to talk |
| Uses appropriate examples/analogies | Abstract without grounding |

#### Scoring Rubric

| Score | Description |
| :---- | :---- |
| **5** | Exceptional clarity; adjusts seamlessly to audience; structures complex information; checks understanding; excellent listening signals |
| **4** | Clear communication; good audience calibration; occasional adjustment needed |
| **3** | Adequate communication; some calibration; may over/under-explain |
| **2** | Communication issues present; poor calibration; tends to confuse or oversimplify |
| **1** | Significant communication barriers; unable to calibrate; poor listening |

### 4.4 Collaboration Assessment Protocol

**Definition**: The ability to work effectively with others toward shared goals.

**IP11 Application**: Trust calibration—weight self-report vs. behavioral evidence.

#### Primary Probes

| Probe | Pattern | What to Listen For |
| :---- | :---- | :---- |
| "Tell me about a successful team project. What was YOUR contribution vs. the team's?" | IP11 | Attribution accuracy |
| "Describe a time you disagreed with a teammate. How did you handle it?" | IP7 | Conflict resolution |
| "How do you handle working with someone whose style is very different from yours?" | MC1 | Perspective-taking |
| "What makes someone easy vs. hard for you to work with?" | MC24 | Self-awareness of preferences |

#### Behavioral Indicators (Observable)

| Strong Collaboration | Weak Collaboration |
| :---- | :---- |
| "We" language with specific individual contributions | Only "I" or only "we" without specifics |
| Acknowledges others' contributions | Takes full credit or no credit |
| Concrete conflict resolution examples | Avoids or escalates conflicts |
| Seeks input before decisions | Makes unilateral decisions |
| Adapts style to team needs | Expects others to adapt |

#### Scoring Rubric

| Score | Description |
| :---- | :---- |
| **5** | Clear examples of effective collaboration; balanced self/team attribution; sophisticated conflict handling; adapts to different working styles |
| **4** | Good collaboration examples; mostly accurate attribution; handles disagreements constructively |
| **3** | Can collaborate; attribution may be biased; conflict handling adequate |
| **2** | Collaboration challenges evident; tends toward solo work; conflict avoidance or escalation |
| **1** | Unable to collaborate effectively; inappropriate attribution; creates conflict |

### 4.5 Initiative Assessment Protocol

**Definition**: The tendency to act proactively without being directed.

**SP8 Application**: Does initiative create value or risk?

#### Primary Probes

| Probe | Pattern | What to Listen For |
| :---- | :---- | :---- |
| "Tell me about something you did that wasn't part of your job description." | Direct | Evidence of proactive action |
| "What's something you identified as a problem and fixed without being asked?" | STAR+ | Problem identification \+ action |
| "How do you balance taking initiative vs. waiting for direction?" | MC38 | Calibration awareness |
| "Describe a time your initiative created problems." | IP7 | Self-awareness of downsides |

#### Behavioral Indicators (Observable)

| High Initiative | Low Initiative |
| :---- | :---- |
| Multiple unprompted contributions | Actions tied to explicit requests |
| Identifies problems AND solutions | Identifies problems, waits for solutions |
| Takes appropriate risks | Risk-averse or reckless |
| Learns from initiative failures | Avoids initiative after setbacks |
| Knows when to check before acting | Always acts OR always asks |

#### Scoring Rubric

| Score | Description |
| :---- | :---- |
| **5** | Strong pattern of valuable proactive action; appropriately calibrated risk-taking; learns from overreach |
| **4** | Good initiative with occasional direction-seeking; creates value proactively |
| **3** | Takes initiative when obvious; may wait for unclear situations |
| **2** | Limited initiative; primarily reactive; needs explicit direction |
| **1** | No initiative evidence; purely reactive; unable to act without direction |

### 4.6 Resilience Assessment Protocol

**Definition**: The capacity to recover from setbacks and maintain effectiveness under pressure.

**MC29 Application**: How do they monitor and recalibrate under stress?

#### Primary Probes

| Probe | Pattern | What to Listen For |
| :---- | :---- | :---- |
| "Tell me about your biggest professional failure." | STAR+ | How they frame and process failure |
| "What did you do immediately after? In the following weeks?" | MC29 | Recovery trajectory |
| "How do you handle sustained high-pressure periods?" | Direct | Coping strategies |
| "What's something you almost gave up on but didn't?" | IP7 | Persistence patterns |

#### Behavioral Indicators (Observable)

| High Resilience | Low Resilience |
| :---- | :---- |
| Discusses failures openly | Avoids failure discussion |
| Describes specific recovery actions | Vague about recovery |
| Separates self-worth from outcomes | Over-identifies with failures |
| Has tested coping strategies | No clear stress management |
| Returns to baseline quickly | Extended recovery periods |

#### Scoring Rubric

| Score | Description |
| :---- | :---- |
| **5** | Discusses significant failures with insight; clear recovery strategies; maintains perspective; demonstrable bounce-back |
| **4** | Can discuss failures; good recovery patterns; some stress management awareness |
| **3** | Handles moderate setbacks; recovery takes longer; limited stress strategies |
| **2** | Difficulty with failure; slow recovery; stress significantly impacts function |
| **1** | Cannot discuss failures; prolonged dysfunction after setbacks; poor stress tolerance |

## Part 5: Top Performer Profiling Protocol

### Purpose

Before assessing candidates, organizations should understand what traits actually differentiate top performers in a specific role. This protocol guides structured interviews with existing high performers to extract trait profiles.

### Top Performer Interview Protocol

#### Phase 1: Context Mapping

| Question | Purpose | Pattern |
| :---- | :---- | :---- |
| "Walk me through a typical week in your role." | Baseline understanding | MC1 |
| "What aspects of the job are most critical to success?" | Identify key demands | SP8 |
| "What makes this role different from how outsiders imagine it?" | Surface hidden requirements | MC24 |

#### Phase 2: Behavioral Extraction

For each of the 6 core traits, ask:

"Think of a specific time your \[TRAIT\] made a real difference in your work..."

  \- What was the situation?

  \- What specifically did you do?

  \- How did it turn out?

  \- What would have happened if you'd approached it differently?

#### Phase 3: Contrast Analysis

| Question | Purpose | Pattern |
| :---- | :---- | :---- |
| "What distinguishes you from average performers in this role?" | Self-identified differentiators | MC24 |
| "Think of someone who struggled in this role. What was missing?" | Failure mode identification | SP8 |
| "What trait would you hire for if you could only pick one?" | Priority signal | MC35 |

#### Phase 4: Hidden Trait Discovery

| Question | Purpose | Pattern |
| :---- | :---- | :---- |
| "What characteristic helps you succeed that most people wouldn't guess?" | Surface non-obvious traits | MC44 |
| "What's a trait you've developed specifically for this role?" | Identify trainable vs. innate | MC38 |
| "What personality type consistently fails in this role?" | Counter-indicators | SP8 |

### Analysis Framework

After interviewing 3-5 top performers, synthesize findings:

| Trait | Frequency Mentioned | Criticality Rating (1-5) | Example Behaviors | Counter-Indicators |
| :---- | :---- | :---- | :---- | :---- |
| \[Trait\] | \[\#/5\] | \[Rating\] | \[Specific behaviors\] | \[What to avoid\] |

**Composite Assessment:**

- Trait Profile Match: \_\_\_% alignment with role profile  
- Confidence Level: L / M / H  
- Recommendation: Strong Hire / Hire / Hold / No Hire

### Calibration Guidelines

To maintain scoring consistency:

1. **Anchor Examples**: Maintain examples of 1, 3, and 5 responses for each trait  
2. **Calibration Sessions**: Review edge cases with other interviewers monthly  
3. **Evidence Requirement**: Each score requires specific behavioral evidence  
4. **Confidence Adjustment**: Mark confidence Low if evidence is thin or contradictory

### Pattern-Based Calibration Questions

When uncertain about a score, apply these metacognitive checks (MC29):

| Question | If Yes | If No |
| :---- | :---- | :---- |
| "Do I have multiple behavioral examples?" | Score may be valid | Lower confidence |
| "Could another interpretation fit the evidence?" | Explore alternatives (MC44) | Proceed |
| "Am I weighting this because it matches my style?" | Adjust for bias (IP11) | Proceed |
| "Would a 6-month manager see what I saw?" | Score likely valid | Reconsider |

## Part 7: Implementation Roadmap

### Phase 1: Role Profile Setup

1. Select target role  
2. Interview 3-5 top performers using Top Performer Protocol  
3. Synthesize trait profile with weights  
4. Define anchor examples for scoring calibration

### Phase 2: Interviewer Training

1. Train on STAR+ methodology  
2. Practice probing techniques for each trait  
3. Calibration sessions with sample responses  
4. Certification: consistent scoring within 1 point

### Phase 3: Candidate Assessment

1. Resume screen for objective requirements (traditional)  
2. Structured interview using trait protocols  
3. Real-time scoring with evidence capture  
4. Calibration discussion for borderline cases

### Phase 4: Continuous Improvement

1. Track hire outcomes vs. assessment scores  
2. Refine trait profiles based on performance data  
3. Update anchor examples with real cases  
4. Recalibrate interviewer scoring drift

## Appendix A: Quick Reference Card

### The 6 Core Traits

| Trait | Key Probe | Primary Signal |
| :---- | :---- | :---- |
| **Curiosity** | "What did you change your mind about?" | Has a list, not one example |
| **Adaptability** | "When your approach wasn't working..." | Speed and quality of pivot |
| **Communication** | "Explain X to a non-expert, then expert" | Calibration ability |
| **Collaboration** | "Your contribution vs. the team's" | Accurate attribution |
| **Initiative** | "Something you did that wasn't your job" | Pattern of proactive value |
| **Resilience** | "Your biggest professional failure" | Recovery trajectory |

### Red Flags

- Cannot provide specific behavioral examples  
- "We" without individual contributions  
- Defensive about failures  
- No questions for interviewer  
- Stories contradict each other  
- All examples are very recent

### Green Flags

- Specific, detailed examples across time  
- Balanced self/team attribution  
- Discusses failures with insight  
- Asks probing questions  
- Consistent patterns across stories  
- Demonstrates trait IN the interview
