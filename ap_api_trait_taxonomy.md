## Comprehensive Personality Characteristic Library with Role-Context Valence Mapping

**Version 1.0 | January 2026** **Author: Kevin Keller, Tucuxi Inc.**

---

## Executive Summary

Personality traits are not universally positive or negative—their value depends entirely on role context. A trait that predicts success in one role may predict failure in another. This document provides:

1. **Expanded Trait Library**: 24 traits organized into 6 categories  
2. **Bidirectional Assessment**: Detecting both presence and absence of traits  
3. **Role-Context Valence Mapping**: When each trait helps vs. hurts  
4. **Counter-Indicator Framework**: Traits that predict failure by role type  
5. **Research Foundation**: Psychological basis for each trait

## Part 2: Complete Trait Definitions

### 2.1 COGNITIVE TRAITS

#### CURIOSITY

**Definition**: The intrinsic drive to explore, understand, and learn beyond immediate requirements.

| Dimension | Low End | High End |
| :---- | :---- | :---- |
| **Spectrum** | Incurious, accepting | Questioning, exploring |
| **Behavioral Markers (Low)** | Accepts information at face value; doesn't ask questions; learns only when required |  |
| **Behavioral Markers (High)** | Questions assumptions; seeks to understand "why"; pursues learning voluntarily |  |

**Role-Context Valence**: | Context | Valence | Rationale | |---------|---------|-----------| | R\&D, Innovation | ✅ HIGH POSITIVE | Discovery requires questioning | | Product Management | ✅ POSITIVE | Understanding user needs | | Assembly Line Work | ⚠️ NEUTRAL | Focus on execution, not exploration | | Highly Regulated Compliance | ⚠️ CONTEXT-DEPENDENT | Questioning rules can be problematic | | Entry-level Execution | ⚠️ MODERATE | Some curiosity good, but need to follow process |

**Counter-Indicator For**: Roles requiring strict protocol adherence without deviation

## Part 3: Counter-Indicators by Role Category

### 3.1 Counter-Indicator Framework

**Definition**: A counter-indicator is a trait that, when present at high levels, predicts failure or poor fit in a specific role type, regardless of whether other traits are strong.

COUNTER-INDICATOR LOGIC:

If Counter\_Indicator\_Trait \= HIGH

   AND Role \= Counter\_Indicated\_Role

Then FLAG\_HIGH\_RISK

Even if all other traits are positive.

### 3.2 Counter-Indicator Matrix

| Role Category | Counter-Indicator Traits | Rationale |
| :---- | :---- | :---- |
| **Safety-Critical** (Surgery, Aviation, Nuclear) | Risk Orientation (HIGH), Creativity (HIGH), Urgency (HIGH) | Proven protocols save lives; innovation is dangerous |
| **Compliance/Audit** | Risk Orientation (HIGH), Creativity (HIGH), Influence (HIGH) | Independence and objectivity essential |
| **Customer Support** | Assertiveness (HIGH), Achievement Orientation (extreme), Independence (HIGH) | Service orientation requires accommodation |
| **Innovation/R\&D** | Consistency (HIGH), Risk Orientation (LOW), Detail Orientation (extreme) | Exploration requires comfort with variation |
| **Executive Leadership** | Independence (extreme), Empathy (LOW), Emotional Regulation (LOW) | Leading requires connection and stability |
| **Entry-level Execution** | Independence (extreme), Initiative (extreme), Strategic Thinking (HIGH) | Need to follow direction while learning |
| **Quality Assurance** | Urgency (HIGH), Risk Orientation (HIGH), Perfectionism (LOW) | Quality requires thoroughness and caution |
| **Fast-paced Startup** | Consistency (HIGH), Perfectionism (HIGH), Ambiguity Tolerance (LOW) | Speed and adaptation essential |
| **Collaborative Research** | Independence (extreme), Assertiveness (extreme), Collaboration (LOW) | Teamwork is essential |
| **Remote/Autonomous** | Independence (LOW), Ambiguity Tolerance (LOW) | Must function without direction |

### 3.3 Counter-Indicator Detection Protocol

**During Top Performer Profiling**:

Questions to surface counter-indicators:

\- "What type of person consistently struggles in this role?"

\- "What trait looks good on paper but predicts failure here?"

\- "Who did you hire that didn't work out, and what was the pattern?"

\- "What's the most common reason people leave or are let go?"

**During Candidate Assessment**:

If trait\_score HIGH on counter\_indicator\_trait for role:

    Probe deeper with:

    \- "Tell me about a time when \[trait\] created problems"

    \- "How do you calibrate when to dial back \[trait\]?"

    \- "Give me an example where less \[trait\] would have been better"

    

    Score adjustment:

    \- Flag for hiring manager review

    \- Note specific concern in assessment

    \- May override positive overall score

### 6.2 Counter-Indicator Checking

class CounterIndicatorChecker:

    def check(self, 

             candidate\_scores: Dict\[str, int\],

             role\_profile: RoleProfile) \-\> List\[CounterIndicatorFlag\]:

        

        flags \= \[\]

        

        for trait\_id, score in candidate\_scores.items():

            trait \= self.trait\_library.get(trait\_id)

            

            \# Check if this trait is a counter-indicator for this role

            if role\_profile.role\_category in trait.counter\_indicator\_for:

                \# Check if score is in dangerous range (usually HIGH)

                if score \>= 4:  \# Configurable threshold

                    flags.append(CounterIndicatorFlag(

                        trait\_id=trait\_id,

                        score=score,

                        role\_category=role\_profile.role\_category,

                        severity=self.calculate\_severity(trait, score, role\_profile),

                        rationale=self.generate\_rationale(trait, role\_profile),

                        recommended\_action=self.recommend\_action(trait, score, role\_profile)

                    ))

        

        return flags

### 6.3 Trait Valence Scoring Adjustment

class ValenceAdjustedScorer:

    def adjust\_score\_for\_valence(self,

                                raw\_score: int,

                                trait: Trait,

                                role\_profile: RoleProfile) \-\> AdjustedScore:

        

        valence \= trait.valence\_by\_role\_category.get(role\_profile.role\_category)

        

        if valence is None:

            \# No specific valence mapping; use raw score

            return AdjustedScore(

                raw=raw\_score,

                adjusted=raw\_score,

                valence\_applied="NEUTRAL",

                adjustment\_rationale="No role-specific valence mapping"

            )

        

        optimal\_min, optimal\_max \= valence.optimal\_range

        

        \# Check if score is in optimal range for this role

        if optimal\_min \<= raw\_score \<= optimal\_max:

            fit\_score \= 5  \# Excellent fit

        elif abs(raw\_score \- optimal\_min) \<= 1 or abs(raw\_score \- optimal\_max) \<= 1:

            fit\_score \= 4  \# Good fit

        else:

            \# Calculate distance from optimal

            distance \= min(abs(raw\_score \- optimal\_min), abs(raw\_score \- optimal\_max))

            fit\_score \= max(1, 5 \- distance)

        

        return AdjustedScore(

            raw=raw\_score,

            adjusted=fit\_score,

            valence\_applied=valence.valence,

            optimal\_range=valence.optimal\_range,

            adjustment\_rationale=f"Role '{role\_profile.role\_category}' optimal range for {trait.name} is {optimal\_min}-{optimal\_max}. Candidate scored {raw\_score}."

        )

### 7.2 Key Meta-Findings

1. **Trait-Performance Relationships Are Moderated by Context** (Barrick & Mount, 2005\)  
     
   - Conscientiousness is positive across most roles  
   - Extraversion matters more for sales/leadership  
   - Context determines which traits predict performance

   

2. **Counter-Indicators Are Real and Predictive** (Hogan & Kaiser, 2005\)  
     
   - "Derailers" predict failure even when strengths are present  
   - Same trait that drives success can cause failure if extreme

   

3. **Optimal Levels Vary by Role** (Le et al., 2011\)  
     
   - Curvilinear relationships exist (too much of good thing)  
   - Context shifts what level is optimal

   

4. **Validity of Assessment Depends on Role-Specific Calibration** (Schmidt & Hunter, 1998\)  
     
   - Generic assessments less predictive than role-specific  
   - Criterion-related validity improves with role alignment
