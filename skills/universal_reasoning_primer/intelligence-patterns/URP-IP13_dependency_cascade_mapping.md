# URP-IP13: Dependency Cascade Mapping

**Type:** Intelligence Pattern (Tier 2)
**Function:** Mapping how dependencies transmit effects through systems
**Domain Independence:** Universal across any interconnected context
**Orchestration Role:** Reveals hidden connections that transmit impacts

## Pattern Definition

### Core Procedure

1. **Dependency Identification**: What depends on what?
2. **Chain Mapping**: How do dependencies chain together?
3. **Cascade Path Tracing**: Where would a change cascade through dependencies?
4. **Criticality Assessment**: Which dependencies are most critical?
5. **Vulnerability Analysis**: Where do cascades concentrate risk?

### Expert Heuristic

> "In complex systems, the first-order effect is rarely the whole story. Map the dependency chains to see where one change becomes many."

## Evidence from Literature

- **Dependency Networks**: Graph-based dependency analysis
- **Cascading Failures** (Perrow): How failures propagate
- **Supply Chain Risk**: Dependency transmission in supply chains
- **Systems Engineering**: Dependency management in complex systems
