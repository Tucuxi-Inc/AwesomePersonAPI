# URP-IP3: Precedent-Based Assessment

**Type:** Intelligence Pattern (Tier 2)
**Function:** Using historical patterns to inform current assessments
**Domain Independence:** Universal across any assessment context
**Orchestration Role:** Grounds abstract assessments in empirical precedent

## Pattern Definition

### Trigger Condition
- Situations requiring judgment where precedents exist
- Decisions where historical patterns are informative
- Need to calibrate estimates against actual outcomes
- Assessment of risks, probabilities, or outcomes
- Validation of abstract analysis against empirical reality

### Core Procedure

1. **Precedent Identification**: What similar situations have occurred? What cases serve as relevant precedents?

2. **Pattern Extraction**: What patterns emerge from precedents? What were actual outcomes?

3. **Applicability Assessment**: How similar is current situation to precedents? What differences affect applicability?

4. **Calibration Application**: How do precedent patterns adjust current assessment? What base rates apply?

5. **Precedent-Limited Reasoning**: Where do precedents not apply? What's novel about current situation?

### Expert Heuristic

> "The best predictor of what will happen is often what has happened in similar situations. Before relying on theory or intuition, ask: what does the historical record say? Be especially skeptical when your assessment differs significantly from precedent patterns."

## Evidence from Literature

- **Reference Class Forecasting** (Kahneman & Lovallo): Using historical base rates improves predictions
- **Case-Based Reasoning** (Kolodner): Past cases inform current judgment
- **Legal Precedent**: Formal system of precedent-based reasoning
- **Statistical Base Rates**: Empirical patterns as prediction foundation

## Decision Criteria

- IF clear precedent exists, THEN weight heavily in assessment
- IF current situation differs materially from precedents, THEN adjust for differences
- IF assessment differs significantly from precedent patterns, THEN require strong justification
- IF precedents are sparse, THEN note increased uncertainty
