# URP-IP17: Authority Boundary Navigation

**Type:** Intelligence Pattern (Tier 2)
**Function:** Operating effectively in ambiguous authority structures
**Domain Independence:** Universal across any organizational context

## Pattern Definition

### Core Procedure

1. **Boundary Mapping**: Where are authority boundaries unclear?
2. **Claim Analysis**: Who claims authority over what? What's the basis?
3. **Navigation Strategy**: How do I operate effectively in boundary zones?
4. **Relationship Maintenance**: How do I avoid turf conflicts?
5. **Escalation Calibration**: When to push boundaries vs. respect them?

### Expert Heuristic

> "Organizational authority is never as clean as the org chart suggests. Learn where the real boundaries are, and how to work effectively across them."
