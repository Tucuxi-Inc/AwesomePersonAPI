# URP-MC41: Assumption Surfacing

**Type:** Meta-Cognitive Pattern (Tier 3)
**Function:** Making implicit assumptions explicit
**Domain Independence:** Universal across all reasoning
**Orchestration Role:** Exposes hidden premises that may be incorrect or contested

## Pattern Definition

### Trigger Condition
Situations involving:
- Conclusions that depend on unstated beliefs
- Plans with implicit requirements
- Arguments with hidden premises
- Decisions based on taken-for-granted assumptions
- Communication where shared assumptions may not exist

### Core Procedure

1. **Assumption Detection**
   - What am I taking for granted?
   - What must be true for this reasoning to work?
   - What am I not questioning?
   - What would someone else question?

2. **Assumption Articulation**
   - State each assumption explicitly
   - What exactly do I believe?
   - How would I phrase this as a testable claim?
   - What are the key premises?

3. **Assumption Classification**
   - Is this a factual assumption (world is X)?
   - Is this a value assumption (X is good)?
   - Is this a definitional assumption (X means Y)?
   - Is this a methodological assumption (approach X is valid)?

4. **Assumption Evaluation**
   - How confident am I in this assumption?
   - What evidence supports it?
   - What would falsify it?
   - Do others share this assumption?

5. **Assumption Handling**
   - IF assumption is solid: proceed but note it
   - IF assumption is uncertain: flag for verification
   - IF assumption is contested: make explicit in communication
   - IF assumption is false: revise reasoning

### Expert Heuristic

> "Every conclusion rests on a hidden foundation of assumptions. The expert doesn't just check the reasoning—they dig down to inspect the foundation. The most dangerous assumptions are the ones you don't know you're making."

## Decision Criteria

**Criterion 1: Critical Assumption Focus**
- IF conclusion depends heavily on assumption, THEN verify carefully
- IF conclusion is robust to assumption, THEN lower priority
- Rationale: Focus on assumptions that matter

**Criterion 2: Assumption Confidence**
- IF highly confident in assumption, THEN can proceed
- IF uncertain about assumption, THEN test or flag
- IF assumption is clearly contested, THEN don't hide it
- Rationale: Handle assumptions according to confidence

**Criterion 3: Communication Requirement**
- IF audience may not share assumption, THEN make explicit
- IF assumption is truly universal, THEN can leave implicit
- Rationale: Shared understanding requires shared assumptions

**Criterion 4: Revision Trigger**
- IF assumption proves false, THEN revise all dependent conclusions
- IF assumption changes, THEN trace implications
- Rationale: Conclusions inherit assumption validity

## Contrast with Naive Approaches

**Naive Approach**: Take assumptions for granted; focus only on explicit reasoning.
**Expert Approach**: Surface and examine assumptions systematically.

**Naive Approach**: Assume others share your assumptions.
**Expert Approach**: Verify shared assumptions; make critical ones explicit.

**Naive Approach**: Defend conclusions; hide shaky premises.
**Expert Approach**: Expose premises; welcome scrutiny of foundations.

**Naive Approach**: Assumptions are fixed background.
**Expert Approach**: Assumptions are testable and revisable beliefs.
