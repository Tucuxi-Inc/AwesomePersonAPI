# URP-MC35: Representation Choice

**Type:** Meta-Cognitive Pattern (Tier 3)
**Function:** Selecting optimal problem or information representation
**Domain Independence:** Universal across all problem-solving and reasoning
**Orchestration Role:** Transforms problems to make solutions more accessible

## Pattern Definition

### Trigger Condition
Situations involving:
- Problems where initial framing seems suboptimal
- Complex information requiring organization
- Abstract concepts needing concretization
- Multi-dimensional situations needing structure
- Failed initial approaches suggesting reframing

### Core Procedure

1. **Current Representation Assessment**
   - How am I currently representing this?
   - What does my current representation make easy/hard?
   - What's hidden by this representation?
   - Is there something awkward about this framing?

2. **Alternative Representation Generation**
   - What other ways could I represent this?
   - What representations work for similar problems?
   - What would a visual look like? A formula? A narrative?
   - What representation would an expert use?

3. **Representation Evaluation**
   - What does each representation reveal/conceal?
   - Which highlights the key relationships?
   - Which aligns with available solution methods?
   - Which fits my cognitive strengths?

4. **Representation Selection**
   - Choose representation that best supports goal
   - Consider multiple representations if useful
   - Note what's lost in chosen representation
   - Plan for representation switching if needed

5. **Translation Execution**
   - Transform to new representation
   - Verify information preserved correctly
   - Confirm new representation is usable
   - Note any insights from translation itself

### Expert Heuristic

> "A problem well-stated is half-solved. If you're struggling, the problem isn't just hard—it might be badly represented. The master problem-solver often wins by choosing a representation that makes the solution obvious."

## Evidence from Literature

### Example 1: Expert Problem Representation
**Context:** Chi et al. (1981) expert-novice studies
**Evidence:** Experts spend more time on problem representation; their representations capture deeper structure
**Insight:** Representation investment pays off

### Example 2: Mutilated Checkerboard
**Context:** Kaplan & Simon (1990) problem-solving research
**Evidence:** Problems can be nearly impossible or trivial depending on representation
**Insight:** Representation dramatically affects solvability

### Example 3: External Representations
**Context:** Zhang & Norman (1994) distributed cognition
**Evidence:** External representations (diagrams, etc.) change what information is accessible
**Insight:** Representation choice includes internal vs. external

## Decision Criteria

**Criterion 1: Solution Path Clarity**
- IF representation makes solution path visible, THEN prefer
- IF representation obscures solution path, THEN reconsider
- Rationale: Representation should serve solution

**Criterion 2: Cognitive Fit**
- IF representation matches cognitive strengths, THEN prefer
- IF representation requires difficult mental operations, THEN consider alternatives
- Rationale: Representation should fit the reasoner

**Criterion 3: Information Preservation**
- IF representation preserves critical information, THEN viable
- IF representation loses important features, THEN may need multiple representations
- Rationale: Representation must capture what matters

**Criterion 4: Method Alignment**
- IF representation aligns with available methods, THEN prefer
- IF representation requires unavailable methods, THEN reconsider
- Rationale: Representation should enable action

## Contrast with Naive Approaches

**Naive Approach**: Work with first representation that comes to mind.
**Expert Approach**: Deliberately consider and evaluate alternative representations.

**Naive Approach**: Struggle with the problem as given.
**Expert Approach**: Reframe the problem to make it more tractable.

**Naive Approach**: Assume the problem is hard.
**Expert Approach**: Consider whether the representation is making it hard.

**Naive Approach**: Single representation throughout.
**Expert Approach**: Use different representations for different phases or aspects.
