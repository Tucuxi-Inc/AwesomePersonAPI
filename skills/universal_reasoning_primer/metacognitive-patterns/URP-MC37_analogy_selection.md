# URP-MC37: Analogy Selection

**Type:** Meta-Cognitive Pattern (Tier 3)
**Function:** Choosing appropriate analogies to guide reasoning
**Domain Independence:** Universal across reasoning and problem-solving
**Orchestration Role:** Imports structure from known domains to illuminate unfamiliar ones

## Pattern Definition

### Trigger Condition
Situations involving:
- Unfamiliar problems requiring structure
- Need to explain complex concepts
- Decisions with limited direct experience
- Novel situations needing reference points
- Teaching or communicating abstractions

### Core Procedure

1. **Target Analysis**
   - What are the key features of this situation?
   - What relationships need to be understood?
   - What aspects are causing difficulty?
   - What kind of structure am I looking for?

2. **Source Retrieval**
   - What do I know that has similar structure?
   - What analogies have I seen used for similar problems?
   - What domains have relevant patterns?
   - What cases could serve as references?

3. **Structural Alignment**
   - What maps from source to target?
   - Are the mapped relationships valid?
   - What doesn't map (disanalogies)?
   - Is the mapping productive or misleading?

4. **Analogy Evaluation**
   - Does this analogy highlight what matters?
   - What does it obscure or distort?
   - Is it generative of new insights?
   - Could it mislead if taken too far?

5. **Application with Limits**
   - Use analogy for what it illuminates
   - Note boundaries of valid mapping
   - Don't overextend beyond structural match
   - Be ready to abandon if misleading

### Expert Heuristic

> "A good analogy illuminates; a bad one misleads. The art is knowing what maps and what doesn't. Every analogy breaks down somewhere—the question is whether it breaks down before or after giving you what you need."

## Evidence from Literature

### Example 1: Structure Mapping Theory
**Context:** Gentner (1983) analogical reasoning research
**Evidence:** Analogies work through structural alignment, not surface similarity
**Insight:** Evaluate analogies by structural, not surface, match

### Example 2: Analogical Problem Solving
**Context:** Gick & Holyoak (1980) studies
**Evidence:** Analogies enable transfer but require recognition of structural similarity
**Insight:** Analogy selection is bottleneck for transfer

### Example 3: Expert Analogy Use
**Context:** Dunbar (1997) studies of scientists
**Evidence:** Scientists frequently use analogies; quality of analogy affects reasoning quality
**Insight:** Analogy selection is a core expert skill

## Decision Criteria

**Criterion 1: Structural Match**
- IF relational structure matches, THEN analogy is useful
- IF only surface features match, THEN analogy is dangerous
- Rationale: Structural match is basis for valid inference

**Criterion 2: Familiarity-Relevance Balance**
- IF source is familiar but structurally different, THEN avoid
- IF source is less familiar but structurally matched, THEN prefer
- Rationale: Familiarity without structure misleads

**Criterion 3: Generativity**
- IF analogy generates new predictions/insights, THEN useful
- IF analogy only restates known, THEN limited value
- Rationale: Good analogies are productive

**Criterion 4: Limit Awareness**
- IF limits are known, THEN can use safely
- IF limits are unknown, THEN use cautiously
- Rationale: All analogies break down; know where

## Contrast with Naive Approaches

**Naive Approach**: Use analogies based on surface similarity.
**Expert Approach**: Select analogies based on structural match.

**Naive Approach**: Trust analogy completely once selected.
**Expert Approach**: Know analogy limits; don't overextend.

**Naive Approach**: Use first analogy that comes to mind.
**Expert Approach**: Generate multiple candidate analogies; evaluate structural fit.

**Naive Approach**: Abandon analogy when any disanalogy is found.
**Expert Approach**: Expect disanalogies; use analogy within valid boundaries.
