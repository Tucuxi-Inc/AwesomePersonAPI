# URP-MC18: Contingency Planning

**Type:** Meta-Cognitive Pattern (Tier 3)
**Function:** Preparing alternative paths before obstacles arise
**Domain Independence:** Universal across all goal-directed activity
**Orchestration Role:** Enables rapid adaptation by pre-computing responses to likely problems

## Pattern Definition

### Trigger Condition
Situations involving:
- High-stakes tasks where failure has significant cost
- Plans with identifiable risk points
- Novel situations with uncertain outcomes
- Dependence on external factors outside control
- Sequential tasks where early failures block later progress

### Core Procedure

1. **Risk Identification**
   - What could go wrong with this plan?
   - What are the key dependencies?
   - What assumptions might prove false?
   - What external factors could disrupt progress?

2. **Probability and Impact Assessment**
   - How likely is each potential problem?
   - How severe would each be if it occurred?
   - Which risks warrant contingency planning?
   - What's the cost of being unprepared?

3. **Contingency Design**
   - What's Plan B if X fails?
   - What would I do if assumption Y proves false?
   - What's the fallback if resource Z isn't available?
   - How can I preserve optionality?

4. **Trigger Definition**
   - What signals that contingency should activate?
   - How will I know primary plan is failing?
   - What's the decision point for switching?
   - How do I avoid switching too early or too late?

5. **Contingency Preparation**
   - What advance work makes contingencies ready?
   - What resources should be reserved for contingencies?
   - How do I keep options open without excessive cost?
   - What can I pre-position for rapid activation?

### Expert Heuristic

> "Hope is not a strategy. For any plan you care about, you should be able to complete the sentence: 'If X fails, I will Y.' If you can't, you're not planning—you're wishing."

## Evidence from Literature

### Example 1: Implementation Intentions
**Context:** Gollwitzer (1999) if-then planning research
**Evidence:** Pre-specifying "if X then Y" dramatically improves goal achievement
**Insight:** Contingency planning can be simple but powerful

### Example 2: Mental Contrasting
**Context:** Oettingen (2012) WOOP research
**Evidence:** Combining positive visualization with obstacle anticipation improves outcomes
**Insight:** Contingency planning complements goal commitment

### Example 3: Scenario Planning
**Context:** Schoemaker (1995) strategic planning research
**Evidence:** Organizations that plan for multiple scenarios adapt more successfully
**Insight:** Contingency planning extends to strategic level

## Decision Criteria

**Criterion 1: Contingency Investment**
- IF risk is high-probability OR high-impact, THEN invest in contingency
- IF risk is low-probability AND low-impact, THEN accept without contingency
- Rationale: Contingency planning has costs; invest where returns justify

**Criterion 2: Contingency Activation**
- IF trigger condition met, THEN execute contingency without hesitation
- IF trigger unclear, THEN gather more information before switching
- Rationale: Pre-planned triggers enable rapid response

**Criterion 3: Option Preservation**
- IF uncertainty high, THEN invest in keeping options open
- IF uncertainty low, THEN can commit more fully to primary path
- Rationale: Options have value under uncertainty

**Criterion 4: Cascading Failures**
- IF failure in one area could cascade, THEN plan for cascade interruption
- IF failures are independent, THEN plan contingencies independently
- Rationale: Cascading failures require system-level contingencies

## Contrast with Naive Approaches

**Naive Approach**: Plan for success; deal with problems when they arise.
**Expert Approach**: Plan for success; prepare for problems before they arise.

**Naive Approach**: Single plan with full commitment.
**Expert Approach**: Primary plan with pre-designed fallbacks at key risk points.

**Naive Approach**: Contingencies are pessimistic; confidence means no backup.
**Expert Approach**: Contingencies enable confident action because downsides are covered.

**Naive Approach**: Think about problems when they happen.
**Expert Approach**: Think about problems when you can act calmly and creatively.

```

  ↓
URP-MC18: Design contingencies
  ↓

  - Contingency design for priority risks
  - Trigger conditions for activation
  - Preparation requirements
  ↓
OUTPUT: Contingency plans + trigger definitions + preparation actions
```
