# URP-MC21: Learning Extraction

**Type:** Meta-Cognitive Pattern (Tier 3)
**Function:** Abstracting transferable knowledge from specific experiences
**Domain Independence:** Universal across all learning contexts
**Orchestration Role:** Converts episodic experience into semantic knowledge for future use

## Pattern Definition

### Trigger Condition
Situations involving:
- Experiences with potential generalizable insights
- Pattern recognition across multiple experiences
- Need to teach or transfer knowledge to others
- Building personal knowledge base
- Preparing for novel situations similar to past experiences

### Core Procedure

1. **Experience Feature Identification**
   - What were the key features of this situation?
   - What variables were present?
   - What was the structure of the problem?
   - What made this case what it was?

2. **Outcome-Feature Linking**
   - Which features led to which outcomes?
   - What were the causal connections?
   - What variations would have changed outcomes?
   - What was necessary vs. incidental?

3. **Abstraction to Principle**
   - What general principle does this illustrate?
   - Can I state this as an if-then rule?
   - What's the abstract structure underlying the specific case?
   - How would I explain this to someone facing a new situation?

4. **Boundary Condition Identification**
   - When does this principle apply?
   - When does it NOT apply?
   - What contextual features gate its relevance?
   - Where are the limits of generalization?

5. **Knowledge Encoding**
   - How do I represent this for future retrieval?
   - What cues will trigger this knowledge?
   - How do I index this in my mental library?
   - How do I avoid overgeneralization?

### Expert Heuristic

> "The power of learning is in the abstraction. One experience can generate a principle that applies to a thousand situations—but only if you extract it properly. The lesson must be abstract enough to transfer but concrete enough to recognize."

## Evidence from Literature

### Example 1: Transfer of Learning
**Context:** Singley & Anderson (1989) transfer research
**Evidence:** Transfer depends on extracting common elements across situations
**Insight:** Abstraction is the mechanism of transfer

### Example 2: Analogical Reasoning
**Context:** Gentner (1983) structure-mapping theory
**Evidence:** Analogical transfer depends on extracting relational structure
**Insight:** Learning extraction must capture structure, not just surface features

### Example 3: Case-Based Reasoning
**Context:** Kolodner (1993) case-based reasoning
**Evidence:** Useful cases are indexed by abstract features for retrieval
**Insight:** Extraction includes creating retrieval paths

## Decision Criteria

**Criterion 1: Abstraction Level**
- IF principle is too specific, THEN limited transfer
- IF principle is too abstract, THEN hard to apply
- SEEK level that maximizes transfer while maintaining applicability
- Rationale: Abstraction is a goldilocks problem

**Criterion 2: Evidence Strength**
- IF principle based on many experiences, THEN higher confidence
- IF principle based on single experience, THEN tentative hypothesis
- Rationale: Induction strength depends on evidence base

**Criterion 3: Boundary Clarity**
- IF boundaries are clear, THEN principle is safe to use
- IF boundaries are unclear, THEN apply cautiously, update with experience
- Rationale: Misapplication is the risk of extracted principles

**Criterion 4: Cue Specificity**
- IF retrieval cues are specific and valid, THEN principle will activate when relevant
- IF cues are vague, THEN may miss opportunities or misfire
- Rationale: Knowledge is useless if not retrieved when needed

## Contrast with Naive Approaches

**Naive Approach**: Remember specific experiences; hope to recognize similar situations.
**Expert Approach**: Extract principles; encode with retrieval cues; apply to novel situations.

**Naive Approach**: Learn the specific lesson; assume it applies exactly to new situations.
**Expert Approach**: Abstract to principles; identify boundaries; adapt to new contexts.

**Naive Approach**: Extract based on surface features.
**Expert Approach**: Extract based on structural/causal features that determine outcomes.

**Naive Approach**: One experience → one lesson.
**Expert Approach**: Integrate across experiences; strengthen or revise principles with evidence.

URP-MC21 functions as an **episodic-to-semantic converter**:

```
INPUT: Experience + reflection output + outcome assessment
  ↓
URP-MC21: Extract learning
  ↓
EVALUATION:
  - Feature identification
  - Outcome-feature linking
  - Abstraction to principle
  - Boundary identification
  - Encoding with retrieval cues
  ↓
OUTPUT: Principles + boundary conditions + retrieval cues
```
