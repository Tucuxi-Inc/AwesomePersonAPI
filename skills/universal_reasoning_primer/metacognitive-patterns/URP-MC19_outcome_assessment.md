# URP-MC19: Outcome Assessment

**Type:** Meta-Cognitive Pattern (Tier 3)
**Function:** Evaluating task results against goals and criteria
**Domain Independence:** Universal across all goal-directed activity
**Orchestration Role:** Closes the feedback loop between planning and execution

## Pattern Definition

### Trigger Condition
Situations involving:
- Completion of any goal-directed task
- Decision points requiring evaluation of prior actions
- Checkpoints in ongoing projects
- Any situation requiring success/failure determination
- Need to report or communicate results

### Core Procedure

1. **Criteria Retrieval**
   - What were the original goals?
   - What did success look like?
   - What were explicit and implicit criteria?
   - What were the constraints?

2. **Outcome Measurement**
   - What was actually achieved?
   - How does outcome compare to goal?
   - Which criteria were met/not met?
   - What's the gap (if any)?

3. **Context Consideration**
   - Did conditions change during execution?
   - Were original goals still appropriate?
   - What unexpected factors affected outcomes?
   - How should context affect judgment?

4. **Multi-Dimensional Assessment**
   - Primary goal achievement?
   - Secondary benefits or costs?
   - Side effects (positive and negative)?
   - Long-term implications?

5. **Summary Judgment**
   - Overall success, partial success, or failure?
   - What's the honest assessment?
   - What would I tell someone else?
   - What would I do differently?

### Expert Heuristic

> "The assessment should answer not just 'did we succeed?' but 'did we succeed at the right thing?' Success on the wrong goal is failure in disguise; failure to achieve a goal that should have changed is not necessarily failure."

## Evidence from Literature

### Example 1: Feedback and Learning
**Context:** Kluger & DeNisi (1996) feedback intervention theory
**Evidence:** Outcome feedback is essential for learning but must be properly structured
**Insight:** Assessment quality affects learning from experience

### Example 2: Goal-Performance Discrepancy
**Context:** Carver & Scheier (1998) self-regulation theory
**Evidence:** Perceived discrepancy between goals and outcomes drives adjustment
**Insight:** Assessment creates the signal for improvement

### Example 3: After-Action Reviews
**Context:** Morrison & Meliza (1999) military learning studies
**Evidence:** Structured outcome assessment dramatically improves learning
**Insight:** Systematic assessment beats informal reflection

## Decision Criteria

**Criterion 1: Success Determination**
- IF outcome meets all primary criteria, THEN success
- IF outcome meets some criteria, THEN partial success requiring analysis
- IF outcome meets no criteria, THEN failure requiring diagnosis
- Rationale: Clear judgment enables appropriate response

**Criterion 2: Context Adjustment**
- IF context changed significantly, THEN adjust criteria before judging
- IF context was stable, THEN apply original criteria strictly
- Rationale: Fair assessment accounts for changing conditions

**Criterion 3: Attribution**
- IF outcome due to approach quality, THEN adjust approach
- IF outcome due to execution quality, THEN adjust execution
- IF outcome due to external factors, THEN adjust expectations
- Rationale: Correct attribution enables correct response

**Criterion 4: Multi-Level Assessment**
- IF primary goal met but secondary costs high, THEN qualified success
- IF primary goal failed but valuable learning occurred, THEN productive failure
- Rationale: Single-dimension assessment misses important outcomes

## Contrast with Naive Approaches

**Naive Approach**: Judge success by feeling; "that went well" without analysis.
**Expert Approach**: Compare outcomes to explicit criteria; measure gaps.

**Naive Approach**: Binary success/failure; no nuance.
**Expert Approach**: Multi-dimensional assessment; partial success recognition.

**Naive Approach**: Assess only the stated goal; ignore side effects.
**Expert Approach**: Assess full outcome including unintended effects.

**Naive Approach**: Forget original goals; assess by current standards.
**Expert Approach**: Retrieve and apply original criteria (adjusted for context).

## Integration with Pattern Tiers

### As Orchestration Operator

URP-MC19 functions as an **outcome-goal comparator**:

```
INPUT: Completed task + outcomes + original goals + context
  ↓
URP-MC19: Assess outcomes
  ↓
EVALUATION:
  - Goal retrieval
  - Outcome measurement
  - Gap analysis
  - Context adjustment
  - Multi-dimensional assessment
  ↓
OUTPUT: Assessment judgment + gap analysis + attribution
```
