# PCP-MC10: Precedent-Based Reasoning

**Type:** Meta-Cognitive Pattern (Tier 3)
**Function:** Leveraging historical patterns to inform current decisions and build strategic credibility
**Domain Independence:** Universal across all professional contexts
**Orchestration Role:** Calibrates risk tolerance and approach based on proven patterns

## Pattern Definition

### Trigger Condition
Situations where:
- A decision involves uncertainty that past experience might reduce
- Strategic position changes might appear opportunistic without context
- New proposals could benefit from proven implementation examples
- Risk calibration requires reference points
- Credibility needs to be established for unfamiliar approaches

### Core Procedure

1. **Precedent Search**
   - What similar situations have we encountered before?
   - What approaches worked (and didn't work) historically?
   - Who was involved and what can they tell us?

2. **Pattern Extraction**
   - What made the precedent successful or unsuccessful?
   - What contextual factors are similar vs. different now?
   - What principles transfer vs. what is situation-specific?

3. **Calibration Application**
   - How does the precedent calibrate our risk tolerance here?
   - What implementation guidance does it provide?
   - What pitfalls should we avoid based on history?

4. **Credibility Leveraging**
   - How does our historical position strengthen our current stance?
   - What narrative connects past and present coherently?
   - How do we demonstrate consistency while enabling evolution?

5. **Documentation for Future**
   - What should we document for future precedent?
   - How do we build institutional memory?

### Expert Heuristic

> "Before inventing a new approach, ask: 'How did we handle this last time?' Past successes provide implementation templates; past failures provide risk calibration."

## Decision Criteria

**Criterion 1: Precedent Applicability**
- IF contextual factors are similar, THEN weight precedent heavily
- IF context differs significantly, THEN extract principles only
- Rationale: Precedents transfer when underlying dynamics match

**Criterion 2: Outcome Quality**
- IF precedent was successful, THEN consider replication
- IF precedent failed, THEN analyze failure mode and avoid
- Rationale: Learn from both successes and failures

**Criterion 3: Credibility Requirements**
- IF position change might appear opportunistic, THEN establish historical context first
- IF proposing novel approach, THEN anchor in precedent where possible
- Rationale: Historical consistency enables strategic flexibility

**Criterion 4: Documentation Imperative**
- IF current approach succeeds, THEN document for future precedent
- IF current approach fails, THEN document lessons learned
- Rationale: Build institutional memory for future decisions

## Contrast with Naive Approaches

**Naive Approach**: Treat every situation as unique; reinvent approaches from scratch.
**Expert Approach**: Search for precedents first; adapt proven approaches to current context.

**Naive Approach**: Change strategic positions without acknowledging history.
**Expert Approach**: Frame position evolution within historical context to maintain credibility.

**Naive Approach**: Rely only on personal memory for precedents.
**Expert Approach**: Systematically access institutional memory and documented precedents.

**Naive Approach**: Apply precedents rigidly without considering contextual differences.
**Expert Approach**: Extract transferable principles while adapting to current context.
