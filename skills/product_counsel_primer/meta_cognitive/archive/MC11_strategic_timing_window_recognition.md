# MC11: Strategic Timing & Window Recognition

**Type:** Meta-Cognitive Pattern (Tier 3)
**Function:** Recognizing time-sensitive opportunities and orchestrating action under time pressure
**Domain Independence:** Universal across all strategic contexts
**Orchestration Role:** Determines when to act, wait, or accelerate based on temporal dynamics

## Pattern Definition

### Trigger Condition
Situations involving:
- External events that create strategic windows (regulatory changes, competitor moves)
- Time pressure from stakeholders or external deadlines
- Tension between speed and quality/process requirements
- Multi-stakeholder approvals needed under time constraints
- Decisions where timing affects outcome as much as content

### Core Procedure

1. **Window Recognition**
   - What external changes create new strategic options?
   - How long will this window remain open?
   - What happens if we miss this timing?

2. **Urgency Assessment**
   - Is this urgency real or perceived?
   - What signals indicate true vs. artificial urgency?
   - Who benefits from the urgency framing?

3. **Urgency-Quality Tension Management**
   - What process requirements cannot be bypassed?
   - What can be parallelized vs. must be sequential?
   - How do we communicate urgency without cutting corners?

4. **Orchestration Under Pressure**
   - Who needs to approve and in what sequence?
   - What can we pre-position before the window opens?
   - How do we maintain quality gates while accelerating?

5. **Strategic Timing Decision**
   - Act now, wait for better moment, or pass entirely?
   - First mover advantage vs. fast follower strategy?
   - What's the cost of delay vs. cost of premature action?

### Expert Heuristic

> "Strategic windows close faster than they open. When external events create legitimate cover for position changes, move quickly to assess implications before positions harden."

## Decision Criteria

**Criterion 1: Window Assessment**
- IF external event creates new options, THEN assess window duration
- IF window is short, THEN prioritize assessment over perfect analysis
- Rationale: Missed windows cannot be recovered

**Criterion 2: Urgency Verification**
- IF urgency claimed, THEN verify through multiple signals
- IF escalation patterns absent, THEN question urgency framing
- Rationale: False urgency leads to poor decisions

**Criterion 3: Process Compression**
- IF true urgency exists, THEN parallelize approvals
- IF quality gates are essential, THEN communicate timeline impact
- Rationale: Speed without quality creates larger problems

**Criterion 4: Timing Optimization**
- IF first mover advantage exists, THEN prioritize speed
- IF fast follower is viable, THEN let others absorb risk
- Rationale: Not all windows require first entry

## Contrast with Naive Approaches

**Naive Approach**: Respond to all urgency claims with the same priority.
**Expert Approach**: Verify urgency through escalation patterns and multiple signals.

**Naive Approach**: Let time pressure bypass quality processes.
**Expert Approach**: Parallelize processes and communicate explicitly about urgency.

**Naive Approach**: Wait for perfect information before strategic pivots.
**Expert Approach**: Recognize that windows close; act on sufficient information.

**Naive Approach**: Assume timing doesn't affect outcomes.
**Expert Approach**: Recognize that when you act often matters as much as what you do.
