# PCP-MC12: Information Architecture & Flow Management

**Type:** Meta-Cognitive Pattern (Tier 3)
**Function:** Strategically managing information flow, integration, and calibration across stakeholders
**Domain Independence:** Universal across all organizational contexts
**Orchestration Role:** Controls what information flows where, when, and in what form

## Pattern Definition

### Trigger Condition
Situations requiring:
- Information distribution to diverse stakeholder groups
- Integration of parallel workstreams or threads
- Calibration of information detail level for different audiences
- Strategic sequencing of disclosure
- Management of information asymmetries

### Core Procedure

1. **Knowledge Mapping**
   - Who knows what relevant to this situation?
   - Who needs to know what to make good decisions?
   - What information asymmetries exist?

2. **Information Architecture Design**
   - What's the optimal structure for this information?
   - How should it be segmented for different audiences?
   - What format best serves each stakeholder's needs?

3. **Flow Sequencing**
   - What order should disclosures follow?
   - What needs internal alignment before external communication?
   - What depends on what?

4. **Density Calibration**
   - How much detail does each audience need?
   - When does more information help vs. hurt?
   - What's the right level of technical depth?

5. **Thread Integration**
   - What parallel workstreams might intersect?
   - Who needs to be aware of related efforts?
   - How do we prevent contradictory communications?

### Expert Heuristic

> "Information is not neutral - its structure, timing, and density shape how it's received and acted upon. Design information flow as carefully as you design the content."

## Decision Criteria

**Criterion 1: Audience Segmentation**
- IF audiences have different needs, THEN structure information for each
- IF single format insufficient, THEN create audience-specific versions
- Rationale: One-size-fits-all communication often fits no one well

**Criterion 2: Sequencing Priority**
- IF internal misalignment possible, THEN align internally first
- IF dependencies exist, THEN respect information sequence
- Rationale: Information order affects reception and outcomes

**Criterion 3: Density Optimization**
- IF more detail might create confusion, THEN simplify
- IF stakeholder needs deep understanding, THEN provide depth
- Rationale: Right information density varies by audience and purpose

**Criterion 4: Integration Monitoring**
- IF parallel workstreams exist, THEN check for integration needs
- IF contradictory communications possible, THEN coordinate
- Rationale: Uncoordinated information creates organizational dysfunction

## Contrast with Naive Approaches

**Naive Approach**: Send the same information to everyone in the same format.
**Expert Approach**: Structure information architecture for different audience needs.

**Naive Approach**: Disclose information as it becomes available.
**Expert Approach**: Sequence disclosures strategically for optimal reception.

**Naive Approach**: Assume more information is always better.
**Expert Approach**: Calibrate density; sometimes less detail serves better.

**Naive Approach**: Work in isolation from parallel efforts.
**Expert Approach**: Actively monitor for and integrate related workstreams.
