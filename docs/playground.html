How the Awesome Person API actually works

An interactive map of the talent assessment engine — from STAR+ probe generation to evidence-weighted scoring to compliance guardrails. Adjust the knobs, click the branches, swap providers. The footer compiles your selections into a prompt you can paste back into the API.

1 · Workflows

Three operating modes share the same evidence-extraction core but differ in framing and phase machine.

2 · STAR+ Probe Loop

After every candidate response, the engine analyzes evidence + STAR completeness + confidence, then routes the next probe. Click a branch to see what triggers it.

3 · Evidence-Weighted Scoring

7 · Compliance Guardrails

Hardcoded fairness checks run in two places: probe-time (before sending to candidate) and analysis-time (when extracting evidence). Selection-rate disparity is monitored separately.

Evidence exclusion rule

When response analysis surfaces evidence containing a prohibited topic, the engine sets the action EXCLUDE_FROM_SCORING and the evidence item is dropped from the trait score — but the rest of the response is still analyzed. This preserves an auditable chain: every excluded item is logged with its trigger.
